# OpenMontage 完整安装调试与部署教程
# 零基础小白版 · 含实测经验与方案选择指南
# 更新日期：2026-06-18

================================================================================
                        第一部分：准备工作
================================================================================

【你需要准备什么】

1. 一台 Windows 电脑（Mac 用户部分步骤不同，会标注）
2. 网络连接
3. 耐心（整个过程约 30-60 分钟）
4. 一个 Agnes API Key（免费注册，后面会讲）

【提前了解的概念】

| 名词 | 简单解释 |
|------|---------|
| Git | 下载代码的工具 |
| Node.js | 运行 JavaScript 程序的环境 |
| Python | 另一种编程语言环境 |
| FFmpeg | 处理视频的工具 |
| API Key | 像"密码"一样，让你能调用 Agnes 的服务 |
| .env 文件 | 存放各种配置密码的文件 |
| Trea | AI 编程助手，帮你写代码和执行命令 |

================================================================================
                        第二部分：安装基础软件
================================================================================

---
步骤 1：安装 Git（下载代码用）
---

【做什么】
Git 是用来从 GitHub 下载 OpenMontage 代码的工具。

【操作步骤】
1. 打开浏览器，访问：https://git-scm.com/download/win
2. 点击下载按钮，下载安装包
3. 双击安装包，一路点击"Next"（全部默认设置即可）
4. 安装完成后，打开 Trea，输入：
   git --version
   如果显示类似 "git version 2.xxx"，说明安装成功

【踩坑提醒】
❌ 坑 1：安装时修改了默认路径
   → 解决：建议保持默认路径 C:\Program Files\Git，不要改

❌ 坑 2：安装时没勾选"Add to PATH"
   → 解决：重新安装，确保勾选"Git from the command line..."

❌ 坑 3：Trea 中输入 git 命令提示"不是内部命令"
   → 解决：重启 Trea 或重启电脑，让环境变量生效

---
步骤 2：安装 Node.js（运行环境）
---

【做什么】
Node.js 是运行 OpenMontage 程序的基础环境。

【操作步骤】
1. 打开浏览器，访问：https://nodejs.org
2. 点击大大的绿色"LTS"按钮下载（推荐长期支持版）
3. 双击安装包，一路点击"Next"（全部默认）
4. 安装完成后，在 Trea 中输入：
   node --version
   如果显示 "v20.xxx" 或 "v22.xxx"，说明成功（需要 18 以上）

【踩坑提醒】
❌ 坑 1：下载了旧版本（v16 或更低）
   → 解决：OpenMontage 需要 Node.js 18+，必须下载 LTS 版

❌ 坑 2：安装路径有中文
   → 解决：确保安装路径全是英文，如 C:\Program Files\nodejs

❌ 坑 3：Trea 中 node 命令无效
   → 解决：重启 Trea，或手动添加环境变量

---
步骤 3：安装 Python（处理脚本用）
---

【做什么】
OpenMontage 部分功能需要 Python 支持。

【操作步骤】
1. 打开浏览器，访问：https://www.python.org/downloads/
2. 点击"Download Python 3.xxx"按钮
3. ⚠️ 重要：安装时务必勾选"Add Python to PATH"
4. 点击"Install Now"
5. 安装完成后，在 Trea 中输入：
   python --version
   如果显示 "Python 3.10.x" 或更高，说明成功

【踩坑提醒】
❌ 坑 1：忘记勾选"Add to PATH"
   → 解决：重新安装，务必勾选！这是最常见的错误

❌ 坑 2：安装了 Python 2.x
   → 解决：OpenMontage 需要 Python 3.10+，必须卸载重装

❌ 坑 3：多个 Python 版本冲突
   → 解决：在 Trea 中使用 python3 命令代替 python 试试

---
步骤 4：安装 FFmpeg（视频处理核心）
---

【做什么】
FFmpeg 是处理视频、音频的核心工具，OpenMontage 离不开它。

【操作步骤】
1. 打开浏览器，访问：https://www.gyan.dev/ffmpeg/builds/
2. 下载 "ffmpeg-release-essentials.7z" 文件
3. 解压到 D:\ffmpeg（或其他英文路径）
4. 将 D:\ffmpeg\bin 添加到系统环境变量 PATH 中：
   - 右键"此电脑" → 属性 → 高级系统设置 → 环境变量
   - 找到 Path → 编辑 → 新建 → 输入 D:\ffmpeg\bin
   - 确定保存
5. 在 Trea 中输入：
   ffmpeg -version
   如果显示版本信息，说明成功

【踩坑提醒】
❌ 坑 1：下载了错误的版本（如 shared 版）
   → 解决：一定要下载 "essentials" 版本

❌ 坑 2：解压路径有中文或空格
   → 解决：路径必须是纯英文，如 D:\ffmpeg，不要 D:\我的工具\ffmpeg

❌ 坑 3：环境变量添加错误
   → 解决：确保添加的是 bin 文件夹路径，不是 ffmpeg 根目录

❌ 坑 4：Trea 中提示找不到 ffmpeg
   → 解决：重启 Trea 或重启电脑，环境变量修改后需要重启生效

================================================================================
                        第三部分：下载 OpenMontage
================================================================================

---
步骤 5：创建项目文件夹并下载代码
---

【操作步骤】
1. 在 D 盘创建一个文件夹：D:\pytools\projects\MyVideoStudio
2. 在 Trea 中输入以下命令：

   cd D:\pytools\projects\MyVideoStudio
   git clone https://github.com/calesthio/OpenMontage.git

3. 等待下载完成（约几分钟，取决于网速）
4. 下载完成后，进入项目目录：

   cd OpenMontage
   dir

   应该能看到 README.md、package.json 等文件

【踩坑提醒】
❌ 坑 1：git clone 提示"无法解析主机名"
   → 解决：检查网络连接，或尝试使用代理/VPN

❌ 坑 2：下载速度极慢或中断
   → 解决：多试几次，或更换网络环境

❌ 坑 3：提示"destination path already exists"
   → 解决：文件夹已存在，先删除旧的 OpenMontage 文件夹再重试

================================================================================
                        第四部分：初始化项目
================================================================================

---
步骤 6：安装项目依赖
---

【操作步骤】
1. 确保在 OpenMontage 目录下：
   cd D:\pytools\projects\MyVideoStudio\OpenMontage

2. 运行安装命令：
   npm install

3. 等待安装完成（可能需要 5-10 分钟，取决于网速）
4. 安装完成后，应该没有红色报错信息

【踩坑提醒】
❌ 坑 1：npm install 报错"Cannot find module"
   → 解决：检查 Node.js 版本是否 18+，不够就升级

❌ 坑 2：安装过程中卡住不动
   → 解决：按 Ctrl+C 中断，重新运行 npm install

❌ 坑 3：提示权限不足
   → 解决：以管理员身份运行 Trea，或在命令前加 sudo（Mac/Linux）

❌ 坑 4：网络超时
   → 解决：设置 npm 镜像源：
     npm config set registry https://registry.npmmirror.com
     然后再运行 npm install

---
步骤 7：检查 make 命令
---

【做什么】
OpenMontage 的初始化脚本可能用到 make 命令，Windows 默认没有。

【操作步骤】
1. 在 Trea 中输入：
   make --version

2. 如果提示"不是内部命令"，说明没有 make
   → 不用慌，Windows 可以用 npm 命令替代

3. 如果有 make 但报错，尝试运行：
   make setup
   如果失败，改用：npm install（步骤6已经做了）

【踩坑提醒】
❌ 坑 1：Windows 用户纠结 make 命令
   → 解决：Windows 不需要 make，npm install 就够了

❌ 坑 2：Mac 用户没有 make
   → 解决：安装 Xcode Command Line Tools：
     xcode-select --install

================================================================================
                        第五部分：配置 Agnes API
================================================================================

---
步骤 8：注册 Agnes 账号并获取 API Key
---

【操作步骤】
1. 打开浏览器，访问：https://platform.agnes-ai.com
2. 点击"Sign Up"，用邮箱注册账号
3. 登录后，进入 API Keys 页面
4. 点击"Create Key"，复制生成的 Key（以 sk- 开头）
5. ⚠️ 妥善保存，不要泄露给他人

【踩坑提醒】
❌ 坑 1：注册后找不到 API Key 页面
   → 解决：通常在 Dashboard 或 Settings 菜单下，找"API"或"Keys"

❌ 坑 2：Key 复制不完整
   → 解决：确保复制完整的 sk- 开头字符串，不要遗漏字符

❌ 坑 3：Key 失效或过期
   → 解决：重新生成一个新的 Key，更新到 .env 文件中

---
步骤 9：配置 .env 文件
---

【操作步骤】
1. 在 OpenMontage 目录下，找到 .env.example 文件
2. 复制一份，重命名为 .env
3. 用记事本打开 .env 文件
4. 添加或修改以下内容：

   # === Agnes 全模态 API ===
   AGNES_API_KEY=sk-你的真实密钥（替换这里）
   AGNES_API_ENDPOINT=https://apihub.agnes-ai.com/v1
   AGNES_IMAGE_MODEL=agnes-image-2.1-flash
   AGNES_VIDEO_MODEL=agnes-video-v2.0
   AGNES_TEXT_MODEL=agnes-2.0-flash

   # === Piper 本地 TTS（免费离线）===
   PIPER_ENABLED=true

   # === 免费背景音乐源 ===
   MUSIC_SOURCE=free_archive

   # === 视频合成 ===
   REMOITION_ENABLED=true
   FFMPEG_PATH=D:\ffmpeg\bin\ffmpeg.exe

5. 保存文件

【踩坑提醒】
❌ 坑 1：.env 文件格式错误
   → 解决：确保没有空格，格式为 KEY=VALUE，不要加引号

❌ 坑 2：FFmpeg 路径错误
   → 解决：Windows 路径用双反斜杠 \\，如 D:\\ffmpeg\\bin\\ffmpeg.exe

❌ 坑 3：Key 包含空格或换行
   → 解决：确保 sk- 后面直接跟字符，没有空格

❌ 坑 4：已有 .env 文件，直接覆盖导致其他配置丢失
   → 解决：先备份原文件，然后追加 Agnes 配置，不要全部替换

================================================================================
                        第六部分：集成 Agnes 到 OpenMontage
================================================================================

【重要说明】
OpenMontage 原生不支持 Agnes provider。当前安装的 OpenMontage 仅原生支持：
FAL / Google / ElevenLabs / OpenAI / xAI / Doubao / Suno / HeyGen / Runway / Pexels / Pixabay / Unsplash

要让 AGNES_API_KEY 真正被 OpenMontage pipeline 使用，需要额外做以下之一：

1. 让 Agnes 通过 FAL 网关路由（如果 Agnes 已接入 fal.ai 生态）
2. 编写自定义 provider 工具（在 tools/image_gen/ 和 tools/video_gen/ 下添加 agnes_image.py 和 agnes_video.py）
3. 作为外部脚本调用（直接在 Python 脚本中读取环境变量发请求）

---
步骤 10：创建 Agnes 图像生成工具
---

在 Trea 中输入：

请帮我在 D:\pytools\projects\MyVideoStudio\OpenMontage 目录下，
在 tools/image_gen/ 文件夹中创建 agnes_image.py 文件，
实现 Agnes 图像生成功能。

要求：
1. 读取 .env 中的 AGNES_API_KEY 和 AGNES_API_ENDPOINT
2. 调用 Agnes Image 2.1 Flash 模型
3. 返回生成的图像文件路径
4. 处理常见的 API 错误（如速率限制、网络超时）

如果 tools/image_gen/ 目录不存在，请先创建。

---
步骤 11：创建 Agnes 视频生成工具
---

在 Trea 中输入：

请帮我在 D:\pytools\projects\MyVideoStudio\OpenMontage 目录下，
在 tools/video_gen/ 文件夹中创建 agnes_video.py 文件，
实现 Agnes 视频生成功能。

要求：
1. 读取 .env 中的 AGNES_API_KEY 和 AGNES_API_ENDPOINT
2. 调用 Agnes Video 2.0 模型
3. 返回生成的视频文件路径
4. 处理常见的 API 错误

---
步骤 12：注册 Agnes 工具
---

在 Trea 中输入：

请帮我在 D:\pytools\projects\MyVideoStudio\OpenMontage 目录下，
修改 tool_registry.py 文件，注册刚才创建的 Agnes 工具：
- agnes_image.py
- agnes_video.py

确保 OpenMontage 的 pipeline 能够识别和调用这两个工具。

【踩坑提醒】
❌ 坑 1：Trea 说找不到 tools/image_gen/ 目录
   → 解决：让 Trea 先创建目录，再创建文件

❌ 坑 2：注册后 Agnes 工具仍不被识别
   → 解决：检查 tool_registry.py 的导入路径是否正确

❌ 坑 3：Agnes API 返回错误
   → 解决：检查 API Key 是否有效，网络是否通畅

================================================================================
                        第八部分：部署方案选择与实测经验
                        【核心章节 · 必读】
================================================================================
================================================================================

OpenMontage 制作视频有四种技术路径，每种都有适用场景和优缺点。
以下基于实测经验总结，帮助你选择最适合自己的方案。

================================================================================
                        方案一：完全零成本路径
              （真实影像素材 + Piper 本地 TTS + 免费音乐）
================================================================================

【原理】
- 图像：从 Pexels / Pixabay / Archive.org / NASA 免费下载
- 视频片段：从上述平台免费下载
- 旁白：Piper 本地离线合成（完全免费）
- 音乐：Free Music Archive / Incompetech（免费）
- 合成：Remotion + FFmpeg 本地渲染

【成本】
$0（完全免费）

【适用场景】
- 纪录片、纪实风格视频
- 历史事件回顾
- 城市人文短片
- 自然科普内容
- 预算为零的个人创作者

【实测经验】

✅ 优点：
- 完全免费，不花一分钱
- 真实影像有历史感和真实感
- 素材来源丰富（Pexels 有数百万免费素材）

❌ 缺点与踩坑：

【坑 1】下载速度极慢
- Pexels / Pixabay 服务器在国外，国内下载速度 50-200KB/s
- 一个 20MB 的视频片段可能要下载 2-5 分钟
- Archive.org 更慢，老影像服务器带宽有限

【坑 2】文件大小不可控
- NASA 素材默认提供 4K 原始版本，单文件 1-2GB
- Archive.org 默认选最高分辨率，可能下载数 GB
- 程序没有自动限制分辨率，容易"爆盘"

【坑 3】素材风格不统一
- 从多个平台下载的素材，色调、清晰度、风格差异大
- 需要后期调色统一，增加制作复杂度

【坑 4】内容不可控
- 搜索"太空"可能返回不相关的素材
- 特定历史画面可能找不到
- 需要反复搜索筛选，耗时

【坑 5】API 速率限制
- Pexels 免费 API 每小时 200 请求
- 频繁搜索可能触发限制，需要等待

【实测案例】
制作"20世纪太空探索"90秒纪录片：
- 计划下载 8 张图像 + 4 段视频
- 实际：NASA 4K 素材单文件 1.35GB，下载卡住
- 总等待时间：>30 分钟（仍未完成）
- 结论：免费方案"能用"但"不好用"

【配置优化（如果坚持用）】
```
# 必须添加的限制配置
MAX_RESOLUTION=1920x1080      # 禁止下载 4K
MAX_FILE_SIZE_MB=20           # 单文件上限 20MB
MAX_TOTAL_SIZE_MB=100         # 总素材上限 100MB
DOWNLOAD_TIMEOUT=30           # 30秒超时
MAX_RETRIES=3                 # 失败重试 3 次
PARALLEL_DOWNLOADS=1          # 单线程下载
SOURCE_PRIORITY=pexels,pixabay,archive_sd  # 优先顺序
```

【适合人群】
- 网络环境好（有代理/VPN）
- 对下载速度不敏感
- 需要真实历史影像
- 愿意手动筛选素材

================================================================================
                        方案二：Agnes 全模态路径
              （Agnes AI 生成所有素材 + Piper TTS + 免费音乐）
================================================================================

【原理】
- 图像：Agnes Image 2.1 Flash 生成
- 视频片段：Agnes Video 2.0 生成
- 旁白：Piper 本地 TTS
- 音乐：免费音乐库
- 合成：Remotion + FFmpeg 本地渲染

【成本】
- Agnes Image：~$0.03-0.15/张（6张约 $0.2-0.9）
- Agnes Video：~$0.10/段（4段约 $0.4）
- Piper：$0（本地免费）
- 音乐：$0（免费库）
- 总计：约 $0.6-1.3/条视频

【适用场景】
- 动画解说视频
- 科幻/概念视频
- 教育动画
- 产品演示
- 需要风格统一的视频

【实测经验】

✅ 优点：
- 零下载等待，素材即时生成
- 风格完全统一，色调一致
- 内容完全可控，想要啥画面就有啥
- 文件大小可控（图像 2-5MB，视频 5-10MB）
- 不受网络波动影响

❌ 缺点：
- 需要 Agnes API Key（免费注册）
- 有轻微成本（但极低）
- 视频生成有速率限制（8 RPM）
- 单次视频片段约 5 秒，长视频需要分段

【实测案例】
制作"区块链工作原理"90秒动画：
- 生成 6 张图像：约 2 分钟，$0.3
- 生成 4 段视频：约 4 分钟，$0.4
- Piper 合成旁白：1 分钟，$0
- 合成渲染：2 分钟
- 总计：9 分钟，$0.7
- 结论：速度快、质量高、成本可接受

【适合人群】
- 追求效率，不想等待下载
- 需要风格统一的商业/教育内容
- 愿意支付极低成本换取体验
- 网络环境一般

================================================================================
                        方案三：商业 API 混合路径
              （FLUX/Kling/ElevenLabs 等付费服务）
================================================================================

【原理】
- 图像：FLUX via Replicate / Midjourney
- 视频：Kling / Runway / Pika
- 旁白：ElevenLabs（高质量）
- 音乐：Udio / Suno（AI生成）
- 合成：Remotion + FFmpeg

【成本】
- 图像：$0.5-2/张
- 视频：$1-5/段
- 旁白：$0.1-0.5
- 音乐：$0.2-1
- 总计：$3-10/条视频

【适用场景】
- 商业广告
- 精品内容
- 对质量要求极高的项目
- 有预算的专业团队

【实测经验】

✅ 优点：
- 质量最高（FLUX 图像、ElevenLabs 语音都是顶级）
- 可控性最强
- 适合商业交付

❌ 缺点：
- 成本高，不适合个人频繁使用
- 需要多个平台账号和密钥
- 管理复杂（多个 API、多个账单）

【适合人群】
- 专业视频制作团队
- 有明确预算的商业项目
- 对质量极度敏感

================================================================================
                        方案四：混合方案（推荐）
              （Pexels/Pixabay 图像 + Agnes 视频 + Piper + 免费音乐）
================================================================================

【原理】
- 图像：Pexels / Pixabay 免费下载（文件小、速度快）
- 视频片段：Agnes Video 2.0 生成（避免下载大文件）
- 旁白：Piper 本地 TTS
- 音乐：免费音乐库
- 合成：Remotion + FFmpeg

【成本】
- 图像：$0（Pexels/Pixabay 免费）
- 视频：~$0.4（Agnes 4段）
- 旁白：$0（Piper 免费）
- 音乐：$0（免费库）
- 总计：约 $0.4/条视频

【适用场景】
- 所有通用场景（最平衡的方案）
- 教程视频
- 社交媒体内容
- 播客视频
- 个人创作

【为什么推荐混合方案？——实测逻辑】

经过对四种方案的实测对比，混合方案是最优解，原因如下：

【对比维度分析】

| 维度 | 方案一零成本 | 方案二Agnes全模态 | 方案三商业API | 方案四混合 |
|------|------------|----------------|------------|----------|
| 成本 | $0 | $0.6-1.3 | $3-10 | $0.4 |
| 速度 | 极慢（下载等待） | 快 | 快 | 快 |
| 质量 | 中（依赖素材质量） | 高（AI生成） | 极高 | 高 |
| 可控性 | 低 | 高 | 极高 | 高 |
| 风格统一 | 差 | 优 | 优 | 良 |
| 网络依赖 | 高（国外服务器） | 中（Agnes API） | 高（多平台） | 低 |
| 复杂度 | 中 | 低 | 高 | 低 |
| 适合小白 | 否（需处理下载问题） | 是 | 否 | 是 |

【混合方案的核心优势】

1. "扬长避短"的素材策略
   - 图像用免费下载：Pexels/Pixabay 的图像文件小（2-5MB）、下载快、质量高
   - 视频用 Agnes 生成：避免下载大文件，风格统一，内容可控
   - 旁白用 Piper：完全免费，本地运行，隐私安全

2. 成本最低的可行方案
   - 比纯免费方案多花 $0.4，但节省 30 分钟以上等待时间
   - 时间成本 > 金钱成本（尤其对新手）

3. 稳定性最好
   - 不依赖单一来源
   - 图像下载失败可 fallback 到 Agnes Image
   - Agnes 失败可用本地素材替代

4. 学习曲线最平缓
   - 不需要处理复杂的下载配置
   - 不需要管理多个商业 API
   - 出错概率最低

【实测案例：混合方案制作"20世纪太空探索"】

配置：
- 图像：Pexels 搜索 "space rocket"、"astronaut"（8张，约 20MB，下载 1-2 分钟）
- 视频：Agnes Video 生成 4 段 5 秒太空场景（约 3 分钟，$0.4）
- 旁白：Piper 中文男声（1 分钟）
- 音乐：Free Music Archive 史诗风格（30 秒下载）
- 合成：Remotion + FFmpeg（2 分钟）

结果：
- 总时间：8-10 分钟
- 总成本：$0.4
- 文件大小：输出约 50MB
- 体验：流畅无卡顿，无需等待大文件下载

【混合方案的 fallback 机制】

```
图像下载流程：
Pexels → 成功？→ 是 → 使用
    ↓ 失败/超时
Pixabay → 成功？→ 是 → 使用
    ↓ 失败
Agnes Image → 成功？→ 是 → 使用
    ↓ 失败
本地默认素材 → 使用
```

这种多层 fallback 确保制作流程不会中断。

================================================================================
                        第十部分：常见问题速查
================================================================================

| 问题现象 | 可能原因 | 解决方法 |
|---------|---------|---------|
| git 命令无效 | 未安装或未加 PATH | 重装 Git，勾选 PATH |
| node 命令无效 | 未安装或版本低 | 安装 Node.js 18+ |
| python 命令无效 | 未加 PATH | 重装 Python，勾选 PATH |
| ffmpeg 命令无效 | 未加环境变量 | 添加 bin 目录到 PATH |
| npm install 失败 | 网络问题 | 换 npm 镜像源 |
| Agnes API 报错 | Key 错误或超限 | 检查 Key，等待重试 |
| 视频渲染失败 | FFmpeg 问题 | 检查路径和版本 |
| 找不到输出文件 | 路径错误 | 检查 output/ 目录 |
| Piper 无中文 | 缺少中文模型 | 让 Trea 下载 zh_CN 模型 |
| 制作到一半卡住 | 内存/CPU不足 | 关闭其他程序，降低分辨率 |
| 下载速度极慢 | 国外服务器 | 切换素材源或改用 Agnes 生成 |
| 下载文件过大 | 未限制分辨率 | 添加 MAX_RESOLUTION=1920x1080 |
| 素材风格不统一 | 多平台混合 | 统一用 Agnes 生成，或后期调色 |
